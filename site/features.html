Sprints 0 → 5Production-ready

Features.

Every component is open-source, containerised, and wired into a single Helm chart. Use the platform as-is or pick the bricks you need.

Sprint 0 — Foundation & Hardware Handshake

Silicon-to-Cloud pipeline over 802.11ah (HaLow), tolerant to multi-second link latency.

HaLow-tolerant ingest

FastAPI POST /api/v1/telemetry/ingest with device-authoritative timestamps and exponential back-off retries on the device side.

MQTT bridge

Mainflux / Mosquitto bridge ships with the chart. Topics use the asset DID for tenant isolation.

PostgreSQL + PostGIS

Time-indexed telemetry, asset registry, energy_shares ledger, low-confidence verification queue.

Sprint 1 — Edge-AI & Meter Digitisation

Analog dial OCR

AI-on-the-edge-device build runs on the ESP32-S3 + camera, returning a confidence score with every reading.

Confidence routing

Readings < 0.80 land in verification_queue and surface in the dashboard for one-click manual approval.

Virtual meters

Sensor DIDs (Decentralized Identifiers) map to virtual meters and to RED II sharing groups.

Sprint 2 — RED II Collective Self-Consumption

Pro-rata allocator

Dedicated red-ii-allocator service, runs every 15 min, writes priced share rows into energy_shares.

Surplus to GSY-e

Leftover surplus auto-posts as P2P offers to GSY-e at the configured markup.

Resident credits

Bill credits visible in the dashboard within minutes of production.

Sprint 3 — VPP Dispatcher & Active Load Control

FlexMeasures forecasting

Hourly Open-Meteo GTI → DC kW upload via POST /sensors/<id>/forecasts/trigger + poll.

SET_LOAD_LIMIT

Price → curtailment ratio → MQTT command. Edge actuators cut EV/heat-pump load to the published limit.

< 3 s end-to-end

Cloud price tick to edge limit. Verified by Prometheus histogram in the engine.

Sprint 4 — Blockchain Settlement & Regulatory Hardening

Energy Web Origin

Per-batch SHA-256 anchor on EW Origin smart contracts. Dry-run mode toggles via Helm value for staging environments.

Guarantees of Origin

NFT minted per asset + batch. Listed via GET /api/v1/certificates.

ZK / Merkle proofs

GET /api/v1/proof/<tx> returns a Merkle-tree proof that the row was included in the batch — no PII leaked.

MiCA-ready audit

Settlement transactions, prices, and DIDs form an immutable audit trail consumable by MiCA-compliant reporting tools.

Local override

Every node firmware honours a local manual override pin (RED II / IEMD residential safety requirement).

GDPR posture

PII stays in the relational DB. Only DIDs and consumption deltas reach the chain.

Sprint 5 — Scale & Resilience

K6 load test

1,000 ingest req/s sustained, p99 < 1.5 s. Lives in tests/k6/ingest_load.js.

Chaos Mesh

Pod-kill on the HaLow gateway and network partition vs. the EW RPC, validating < 30 s RTO.

Kubernetes HPA

Horizontal autoscaling on CPU for the ingest tier — 2 → 20 replicas during peak VPP events.

One-click Helm

Cross-border deployable: helm upgrade --install eco-vpp ./helm/eco-vpp.
