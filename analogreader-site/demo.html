What happens in production

1. Photo uploads to POST /api/v1/readings/ocr (multipart). The endpoint validates EXIF, hashes the image, and queues an OCR job.

2. The on-device or cloud model returns digits + confidence within 2–4 s.

3. If confidence ≥ 0.80, the reading is stored and chartable. Otherwise it lands in a verification queue.

4. A push notification (or webhook) fires for any subscribed integrations.

Wire this to a real backend

// in assets/app.js, replace mockOCR with:
async function ocr(file, kind) {
  const fd = new FormData();
  fd.append('photo', file);
  fd.append('kind', kind);
  const r = await fetch('/api/v1/readings/ocr', { method: 'POST', body: fd });
  return r.json();
}
