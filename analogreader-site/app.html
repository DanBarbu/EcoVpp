The app.

This is the actual reusable UI module — a single <analog-reader> Web Component you can drop into any page. Capture or upload a meter photo, get a confidence-scored reading, and watch the local history build up. It runs in mock mode here; point the endpoint attribute at a real OCR API and it's production-ready.

Embed it anywhere

Two lines. No build step, no framework, no npm.

<script src="assets/analog-reader.js"></script>
<analog-reader
   endpoint="/api/v1/readings/ocr"
   kinds="power,gas,water">
</analog-reader>

Drop the endpoint attribute and it runs in offline mock mode (what you see on the left).

It emits events

const el = document.querySelector('analog-reader');
el.addEventListener('reading', e =>
  console.log(e.detail));        // {value,unit,confidence,cost,…}
el.addEventListener('reading:review', e =>
  notifyOps(e.detail));          // confidence < 0.80
el.exportCSV();                  // programmatic export

Wire to the real OCR

The endpoint just needs to accept multipart {photo, kind} and return { value, unit, confidence }. The EcoVpp webhook-receiver service speaks this contract — see the API page.

What this module does

⚡

Multi-meter

Electricity, gas, water — switch with the dropdown. Each has its own unit, tariff, and chart colour.

🎯

Confidence gating

Readings under 0.80 are flagged and fire a reading:review event for your ops queue.

💾

Local history

Readings persist in localStorage so the demo survives reloads. Disable with storage="".

📊

Built-in chart

Auto-renders a bar chart of recent readings for the selected meter.

📥

CSV export

One click (or .exportCSV()) for a utility-ready CSV.

🔌

Offline-safe

If the live endpoint errors, it transparently falls back to mock so the UI never dead-ends.

Same engine as EcoVpp

This module is the consumer face of the open-source EcoVpp meter-digitisation pipeline. Run the EcoVpp stack, point endpoint at its webhook-receiver, and you have a production meter app with on-chain settlement behind it.
