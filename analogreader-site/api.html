Developer API.

Push readings in, pull them out, listen for events. Bearer-token auth, JSON throughout.

Authentication

curl -H "Authorization: Bearer YOUR_TOKEN" https://api.analogreader.online/v1/me

Tokens are managed in Settings → API. Pro plans get one token, Team plans get one per apartment, Enterprise gets fine-grained scopes.

Submit a reading (OCR)

curl -X POST https://api.analogreader.online/v1/readings/ocr \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "photo=@meter.jpg" \
  -F "kind=power" \
  -F "meter_id=apt-1-power"

→ 202 Accepted
{
  "job_id": "rd_01HX...",
  "status": "queued"
}

Submit a reading (manual / device)

curl -X POST https://api.analogreader.online/v1/readings \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"meter_id":"apt-1-power","value":48213,"unit":"kWh","timestamp":"2026-05-08T18:00:00Z"}'

→ 201 Created
{
  "id": "rd_01HX...",
  "meter_id": "apt-1-power",
  "value": 48213,
  "unit": "kWh",
  "confidence": 1.0,
  "created_at": "2026-05-08T18:00:01Z"
}

List readings

GET /v1/readings?meter_id=apt-1-power&from=2026-05-01&to=2026-05-08

→ 200 OK
{
  "readings": [
    {"id":"rd_…","value":48213,"unit":"kWh","timestamp":"2026-05-08T18:00:00Z","confidence":0.97},
    ...
  ]
}

Webhooks

Configure a webhook URL in Settings → API → Webhooks. We POST JSON when:

  • reading.created — a new OCR or manual reading was stored.
  • reading.review_required — confidence < 0.80, awaiting your confirmation.
  • alert.leak_suspected — non-zero baseline overnight consumption detected.
  • alert.budget_exceeded — projected bill exceeds your configured budget cap.

SDKs

First-party SDKs ship in the EcoVpp repo: Python, JavaScript/TypeScript, MicroPython (ESP32), and a Home Assistant integration.

Rate limits & quotas

Pro: 600 OCR/day, 5,000 reads/day. Team: 6,000 / 50,000. Enterprise: custom. Bursts up to 10 req/s on every plan.
